# FEAT-260511.4: Union-AABB cluster discovery

## Status: Done
## Version: v2.5.4
## Found In: `NanobotSystem.Scanning.Block.cs` (`AsyncAddBlocksOfBox`)

## Description

`AsyncAddBlocksOfBox` (the BoundingBox-mode discovery path) queried
`SharedEntityCache.GetEntitiesInBox` with the **coordinator's** AABB only —
even when the cluster had many members spread across a large physical region.

Result: cluster members positioned outside the coordinator's AABB couldn't
see any external target grids, because grid *discovery* happened entirely in
the coordinator's box. The block-level `IsInRange` filter then trimmed each
member to its own OBB downstream, but that filter only sees blocks that made
it into the cluster's candidate list — and the candidate list only contained
grids the coordinator's query returned.

Reported by an admin with 58 BaRs spread across multiple grids: only the
target grids near the coordinator were being worked, while distant grids
visible to other members went untouched.

## Implementation

`AsyncAddBlocksOfBox` now unions the AABBs of all cluster members before
calling `GetEntitiesInBox`:

```csharp
var discoveryBox = areaBoundingBox;
if (_ClusterMemberAreaBoxes != null && _ClusterMemberAreaBoxes.Count > 1)
{
    for (int i = 0; i < _ClusterMemberAreaBoxes.Count; i++)
    {
        var memberAABB = _ClusterMemberAreaBoxes[i].GetAABB();
        discoveryBox = BoundingBoxD.CreateMerged(discoveryBox, memberAABB);
    }
    clusterUnionUsed = true;
}
var entityInRange = SharedEntityCache.GetEntitiesInBox(ref discoveryBox);
```

`_ClusterMemberAreaBoxes` is already snapshotted at the start of
`AsyncClusterScan` for the existing per-member distance-sort path; we just
reuse it for discovery.

The per-block filter downstream (`AsyncAddBlockIfWeldTarget` /
`IfGrindTarget`) still uses `skipRangeCheck = true` for multi-member
clusters — block-level inclusion happens later, per-member, in
`ApplyClusterResultToSelf` via `IsInRange(ref areaOrientedBox)`. So the
broader discovery doesn't change *what a BaR ends up welding/grinding*,
only *which grids get a chance to be considered*.

## Profiler output

`AsyncAddBlocksOfBox` now emits `clusterUnion=True/False;memberBoxes=N` so
it's possible to verify the union path fires for multi-member clusters.

## Trade-off

Larger discovery query → potentially more entities returned by
`GetEntitiesInBox` → more enumeration work in the background scan. Measured
on a 58-BaR cluster, `AsyncAddBlocksOfBox` jumped from low single-digit ms
to ~29 ms avg / 65 ms max — still well within background-thread tolerance
and amortised over the per-scan-interval cadence (default 10 s).

Single-member clusters (`memberCount <= 1`) skip the union entirely and
behave exactly as before.
