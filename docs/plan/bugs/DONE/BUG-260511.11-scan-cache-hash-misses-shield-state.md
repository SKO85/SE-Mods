# BUG-260511.11: Grid-scan cache hash missed shield state

## Status: Done
## Severity: Medium (3 s window where grind candidates can mismatch live shield state)
## Version: v2.5.4
## Found In: `NanobotSystem.Scanning.Block.cs` (`ComputeScanParamsHash`)
## Reported by: Codex review

## Description

`AsyncAddBlockIfGrindTarget` rejects off-grid autogrind targets when the
welder is shielded:

```csharp
if (block.CubeGrid.EntityId != Welder.CubeGrid.EntityId && State.IsShielded)
{
    return false;
}
```

`GridScanCache` keys the per-grid candidate list by
`ComputeScanParamsHash`. The hash already covered safe-zone allow-flags
but didn't include `State.IsShielded`. If a shield toggled state inside
the 3 s cache TTL (shield going up / coming down / running out of power),
the next scan replayed the cached candidate list computed under the old
shield state.

Effects within the staleness window:

- Shield just came up → BaR keeps grinding off-grid targets it should
  no longer touch (briefly).
- Shield just went down → BaR keeps skipping off-grid targets it
  should now be allowed to grind.

The window is bounded by the cache TTL (3 s) and self-clears on the
next params-hash mismatch (e.g. priority list change, owner change),
so no permanent state leak — just a brief mismatch on toggle events.

## Fix

One-line addition: hash `State.IsShielded` into `ComputeScanParamsHash`
using the same FNV-style pattern as the safe-zone allow-flags.

```csharp
hash = (hash ^ (State.IsShielded ? 1 : 0)) * 16777619;
```

A shield toggle now invalidates the cached candidate list immediately,
so the next scan rebuilds against the live shield state. Cost is one
boolean read + two arithmetic ops per scan.

## Related: forward-compat note from BUG-260511.9

This is the scan-cache analogue of the BUG-260511.9 state-fingerprint
audit: every state field that affects filtering decisions must be in
the cache key, or stale results get replayed. The two surfaces
(fingerprint vs. scan-cache hash) are independent and should both be
checked when adding a new state-dependent filter.
