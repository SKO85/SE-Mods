# BUG-260511.12: AllowedWorkModes lacked migration for deprecated `GrindIfWeldGetStuck` bit

## Status: Done
## Severity: Medium (legacy worlds: empty terminal combobox + per-block `WorkMode` outside allow-mask)
## Version: v2.5.4
## Found In: `Models/SyncBlockSettings.cs` (per-block clamp), `Models/SyncModSettings.cs` (`ValidateAndClamp`)
## Reported by: Codex review

## Description

`WorkModes.GrindIfWeldGetStuck` was deprecated several versions back. The
per-field `WorkMode` setter migrates the value to `WeldBeforeGrind` (and
the network receive path does the same), but **`AllowedWorkModes`** in
`Mod.Settings.Welder` was never migrated. The clamp inside
`SyncBlockSettings.SetDefaultsToValidValues` only checked for
`WeldBeforeGrind` / `GrindBeforeWeld` in the fallback chain.

Failure case: a world upgraded from a long-ago version that still has
`AllowedWorkModes = GrindIfWeldGetStuck` (and nothing else):

1. Per-block `WorkMode` is migrated to `WeldBeforeGrind` at value-set time.
2. The clamp sees `(AllowedWorkModes & WorkMode) == 0` (mask has only
   the deprecated bit, `WorkMode` is `WeldBeforeGrind`).
3. `WorkModeDefault` similarly isn't in the mask.
4. Inner fallback chain checks `WeldBeforeGrind`, `GrindBeforeWeld` — both
   absent — and **does nothing**.
5. `WorkMode` stays `WeldBeforeGrind`, outside the configured allow-mask.

Combined with the terminal combobox builder (`Terminal/ComboBoxes.cs:133-140`)
which only renders entries for `WeldBeforeGrind` / `GrindBeforeWeld` /
`WeldOnly` / `GrindOnly`, the combo box shows **nothing** and the player
can't select any mode.

Same shape applies to any allow-mask containing only `WeldOnly` or only
`GrindOnly` — the fallback chain was incomplete in those cases too.

## Fix

Two-layer fix:

### 1. Canonicalise in `SyncModSettings.ValidateAndClamp` (load + receive path)

Fold `GrindIfWeldGetStuck` into `WeldBeforeGrind` and clear the
deprecated flag. Also migrate `WorkModeDefault` if it's the deprecated
value. Empty `AllowedWorkModes` (post-migration) gets the full default
mask restored, so a hand-edited `ModSettings.xml` with only the
deprecated bit can't break the terminal combobox.

```csharp
if ((settings.Welder.AllowedWorkModes & WorkModes.GrindIfWeldGetStuck) != 0)
{
    settings.Welder.AllowedWorkModes =
        (settings.Welder.AllowedWorkModes & ~WorkModes.GrindIfWeldGetStuck) | WorkModes.WeldBeforeGrind;
}
if (settings.Welder.WorkModeDefault == WorkModes.GrindIfWeldGetStuck)
    settings.Welder.WorkModeDefault = WorkModes.WeldBeforeGrind;
if (settings.Welder.AllowedWorkModes == 0)
    settings.Welder.AllowedWorkModes =
        WorkModes.WeldBeforeGrind | WorkModes.GrindBeforeWeld | WorkModes.WeldOnly | WorkModes.GrindOnly;
```

This is the canonical surface (runs on every load and every settings
broadcast) — once the value lands here, downstream callers see a clean
mask.

### 2. Extend the per-block clamp fallback chain (defence-in-depth)

```csharp
var allowed = Mod.Settings.Welder.AllowedWorkModes;
if ((allowed & WorkModes.GrindIfWeldGetStuck) != 0)
{
    allowed = (allowed & ~WorkModes.GrindIfWeldGetStuck) | WorkModes.WeldBeforeGrind;
}
if      ((allowed & Mod.Settings.Welder.WorkModeDefault) != 0) WorkMode = …;
else if ((allowed & WorkModes.WeldBeforeGrind) != 0)           WorkMode = WeldBeforeGrind;
else if ((allowed & WorkModes.GrindBeforeWeld) != 0)           WorkMode = GrindBeforeWeld;
else if ((allowed & WorkModes.WeldOnly) != 0)                  WorkMode = WeldOnly;
else if ((allowed & WorkModes.GrindOnly) != 0)                 WorkMode = GrindOnly;
```

Two improvements: (a) handles `GrindIfWeldGetStuck` locally even if a
runtime-mutated mask still has it, and (b) extends fallback to cover
`WeldOnly` / `GrindOnly` so an admin choosing one of those as the
only allowed mode doesn't strand the per-block `WorkMode`.
