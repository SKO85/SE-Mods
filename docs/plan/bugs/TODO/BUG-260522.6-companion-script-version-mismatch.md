# BUG-260522.6: Companion script version still shows v2.5.3

## Status: Open
## Severity: Low (cosmetic / version-sanity-check confusion)
## Version: v2.5.4
## Found In: `SKO-Nanobot-BuildAndRepair-System-Script/metadata.mod:3`, `SKO-Nanobot-BuildAndRepair-System-Script/Script.cs:1`
## Reported by: Copilot (copilot-pull-request-reviewer) — PR #136 review

## Description

Two places in the companion script package still declare v2.5.3 even
though the main mod was bumped to v2.5.4:

- `SKO-Nanobot-BuildAndRepair-System-Script/metadata.mod` line 3:
  `<ModVersion>2.5.3</ModVersion>`
- `SKO-Nanobot-BuildAndRepair-System-Script/Script.cs` line 1:
  `// Version: v2.5.3 - 16.04.2026 -`

Players who paste this script into a Programmable Block can read the
header / metadata to verify they have the matched companion build —
the stale version makes that check misleading.

The companion script is conceptually decoupled from the mod (it works
against the mod's public scripting API, not internal types), so the
version mismatch is not a functional bug, just a hygiene one. If the
two are meant to ship together as a "v2.5.4 release," both should
read v2.5.4.

## Fix

1. Bump `metadata.mod` to `<ModVersion>2.5.4</ModVersion>`.
2. Bump `Script.cs` header to `// Version: v2.5.4 - <today's date>`.
   Also drop the trailing `-` (looks like an unfinished comment).
3. Consider adding to the release-checklist that the script package's
   version mirrors the main mod's whenever it ships in the same PR.

## Verification

Visual inspection — no build or runtime impact since the script is
loaded by SE as a programmable-block script, not compiled into the
mod DLL.
