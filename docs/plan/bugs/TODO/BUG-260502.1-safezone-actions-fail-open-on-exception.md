# BUG-260502.1: Safe-zone permission lookup fails open on exception
## Status: Open
## Severity: Medium
## Version: v2.5.4
## Found In: REVIEW-260502.1 (full Codex code review), finding #1

## Description

`SafeZoneHandler.GetActionsAllowedForSystem` (`Handlers/SafeZoneHandler.cs:449`) initialises the response with `IsGrindingAllowed = IsWeldingAllowed = IsBuildingProjectionsAllowed = true` and uses a try/catch that logs and returns those defaults on any exception (line 475). If anything inside the safe-zone lookup throws (e.g. a transient engine state issue, a CastProhibit failure, a missing entity), the BaR is silently granted full action rights for that tick — safe-zone restrictions are bypassed.

Note: the `CastProhibit<T>` pattern itself (line 419) is the canonical SE-modding shape for invoking `IsActionAllowed` and is *not* the bug; many public mods use it. The defect is the fail-open default on the catch path.

## Root Cause

Defensive default chosen for backward compatibility (returns "all allowed" so callers don't gate on missing data). Combined with a broad catch, any unexpected failure silently drops protection.

## Fix

Flip the initialisation in `GetActionsAllowedForSystem` so the catch returns a fail-closed (all `false`) result when any exception bubbles up:

- Initialise `ActionsState` with all flags `false` only inside the catch path (or after a try succeeds without finding a zone, restore the all-`true` default on the success path).
- Easier alternative: keep the all-`true` default on the **no-zone** path (lines 460-461), but in the catch, log + return `IsGrindingAllowed = IsWeldingAllowed = IsBuildingProjectionsAllowed = false`. That preserves the no-zone-means-allowed semantics while making exceptions fail-closed.

Caller behaviour to verify: BaRs with `false` permission flags will be skipped at scan time (Scanning.Block.cs uses these via `State.SafeZoneAllowsXxx`). Acceptable degradation while a transient SE error clears.

## Verification

Inject a forced exception in the safe-zone path and confirm welding/grinding/building all stop until the next clean call refreshes state.
